### 1.6 Dedicated ErrorBoundary

**What:** A React error boundary component that catches render-time exceptions and displays a contextual fallback UI instead of a blank screen.

**Where it would live:**
- Component: `packages/design-system/src/components/ErrorBoundary.tsx`
- Integration: wrap route segments in `apps/web-app/src/app/[locale]/layout.tsx` and any data-heavy leaf pages; also candidate for `apps/web-clinician` and `apps/web-enterprise`

**Prerequisites:**
- Should log caught errors to the `app` log stream (not `audit`); requires a client-side logging shim that calls the API logging endpoint
- T3/T4 crisis paths must never rely on this component — crisis UI uses deterministic static rendering and must never throw

## Section 3 — Known correctness issue (i18n gap)

This section records a pre-existing bug found during the v1 review. It is **not** a feature deferral — it is a correctness issue that blocks `fr`, `ar`, and `fa` locale release for the Dashboard page.

### 3.1 Dashboard page has hardcoded English strings

**File:** `apps/web-app/src/app/[locale]/page.tsx`

**What:** The following strings are rendered as hardcoded English literals rather than being fetched via `useTranslations`:

- `"Streaks"`
- `"Insights"`
- `"All patterns"`
- `"Daily tip"`
- The urge-wave tip body copy

**Impact:** Any user whose locale is `fr`, `ar`, or `fa` sees English labels on the Dashboard. Arabic and Persian are RTL — mismatched language in an LTR-rendered string also breaks layout expectations.

**Fix required:**

1. Add catalog keys to `packages/i18n-catalog/src/catalogs/en.json` (source of truth), e.g.:
   - `dashboard.section.streaks`
   - `dashboard.section.insights`
   - `dashboard.section.allPatterns`
   - `dashboard.section.dailyTip`
   - `dashboard.urgeTip.body` (or a structured key per tip if tips are dynamic)

2. Add parallel keys to `fr.json`, `ar.json`, `fa.json` with `_meta.status: "draft"` until native reviewer sign-off. Do **not** machine-translate — leave keys in `"draft"` state and let the render layer fall back to `en` until review is complete (per the no-MT policy in `Docs/Technicals/15_Internationalization.md`).

3. Wire `page.tsx` to `useTranslations('dashboard')` and replace every hardcoded string.

4. Rebuild `@disciplineos/i18n-catalog` before running `web-app` locally:
   ```bash
   pnpm --filter @disciplineos/i18n-catalog build
   pnpm --filter @disciplineos/web-app dev
   ```

**Blocks:** `fr`, `ar`, and `fa` locale release for the Dashboard route until resolved.
